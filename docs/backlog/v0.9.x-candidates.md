## v0.9.1 candidates — Phase 4 review carry-over (P2/P3 hygiene)

Small mechanical fixes deferred from Phase 4 fix-loop. Each is < 50 LOC.

### ADR-0016 Validator Registry (PR #10)
- **REV/P2** — `ctx.productConfig` shallow-frozen but reference is mutable; document or deep-freeze copy. `core/manifest/validator.mjs:85`.
- **REV/P2** — `_resetRegistryForTests` exported from `core/registry.mjs`; convention (`_` prefix) is sole guard against downstream products importing core directly. Move to a separate `registry.testutil.mjs` OR add a `process.env.NODE_ENV === "test"` guard.
- **REV/P2** — Cascade behavior: rules run on subjects already flagged by built-in checks. Either document or skip when prior finding targets same `section:id`.
- **REV/P2** — `withRegistryReset` test pattern at `validator.test.mjs:155-160` is fragile under `node:test` concurrency; lock per-call dependency injection OR set `concurrency: false`.
- **REV/P3** — `${rule.appliesTo}s` implicit pluralization (`validator.mjs:103`); future `policy` → `policys` silent miss. Use explicit map.

### ADR-0019 TOCTOU (PR #11)
- **REV/P2** — `withPatchedLockSync` test helper global monkey-patches `lockfile.lockSync`; thrown `injectDuringLock` can leak the patch. Scope by `t.before`/`t.after` hooks.
- **REV/P2** — FAT32/exFAT degradation (Stats.ino === 0) acknowledged in ADR-0019 but no test. Add a smoke test that mocks `lstatSync.ino = 0` and asserts symlink-only behavior.
- **REV/P3** — `verifyPostLockIntegrity` throws plain `new Error()` with `.code`; introduce `class ToctouError extends Error` for ADR-0014 envelope serialization fidelity.

### ADR-0017 deactivate (PR #12)
- **REV/P2** — Seam-collapse asymmetric trailing newline (when fence at file-end). After splice, file may end without `\n`. Normalize to always end with exactly one `\n` when content remains.
- **REV/P2** — No concurrent-lock test for `removeAmbientFence`. Add ELOCKED → ERR_LOCKED case (pre-creates lock dir).
- **REV/P2** — `deactivateCommand` does not validate `scope`; only `runDeactivate` does. Same gap exists in `activateCommand`. Paired fix.
- **REV/P2** — `createBufferedLogger` duplicated verbatim from `activate.mjs`. Hoist into shared module (`cli/commands/_logger.mjs` or similar).

### ADR-0018 CI bless (PR #8)
- **REV/P2** — Test fixture default `{GITHUB_ACTIONS: "true", NEXEL_CI_BLESSED: "true"}` masks future signal requirements. Prefer marker fixtures for happy-path tests; reserve env-only paths for dedicated ADR-0018 cases.
- **REV/P2** — `release.yml.test.mjs` substring-asserts the bypass-scan needles; does not validate the regex anchors. Test against commit fixtures pass/fail directly.
- **REV/P2** — `readBlessMarker` has `existsSync` + `lstatSync` micro-TOCTOU. Use single `lstatSync` + ENOENT catch (mirrors ADR-0019 `snapshotPreLockStats` pattern after Phase 4 fix).

## v0.10.0 candidates — ideation ranked survivors

From `docs/ideation/2026-05-28-...md` post-grill ranking. v0.9.0 covered
Idea 2 (Validator Registry portion) + Idea 5 (TOCTOU partial). 6 remain.

### Rank 1 — Idea 4a: Plugin Runtime 横向扩展 (Conf 90%, Complexity Low-Med)
- **IDE** — Add `scripts/installer/adapters/claude-plugin.mjs` + `codex-plugin.mjs` mirroring existing `opencode-plugin.mjs`. Same subpath-export pattern (ADR-0009).
- Recommended next target — highest confidence, low risk, fills coverage gap (currently install-plane/activation-plane split only OpenCode-implemented).
- Likely ADR-0020.

### Rank 2 — Idea 5: State schema-migration ladder (Conf 85%, Complexity Med)
- **IDE** — `state.json` adds `stateSchema: N`; kernel walks registered `v(N) → v(N+1)` pure-function migrations on startup.
- v0.9.0 sidestepped state-schema changes; future CAS state / host-registry will need this ladder. Build now while pre-1.0 unfreeze budget cheap.
- Likely ADR-0021.

### Rank 3 — Idea 3: Optional staged content pipeline SPI (Conf 85%, Complexity Med)
- **IDE** — SPI v1.3 adds optional `contentPipeline?: Stage[]`. Coexists with v1.2 `transformAssetContent?` (auto-promotes single hook to 1-stage pipeline). Additive minor per ADR-0010.
- Unlocks: third-party stage packs, self-improve hook landing point, lint as a stage.
- Likely ADR-0022.

### Rank 4 — Idea 1: Skill Vendoring + `@nexel/vendor` 姊妹包 (Conf 85%, Complexity Med, kernel zero changes)
- **IDE** — Sister package `@nexel/vendor` declarative `vendorFrom` + `.vendored.json` sidecar + `vendor {pull|diff|fork}` CLI. Kernel zero changes (consumes public API only).
- Blocked on: kernel public API publish (ADR-0005 v1.0 clock) — but design + scaffold can land before adoption.

### Rank 6 — Idea 6: Capability handshake + conformance + doctor probes (Conf 80%, Complexity Med)
- **IDE** — Three-piece: adapter `capabilities()`; `@nexel/adapter-conformance` published package; doctor `probes()` SPI.
- Enables SPI evolution via capability bits instead of break-all-adapters.

### Rank 7 — Idea 7: Manifest from filesystem 派生 (Conf 80%, Complexity Med)
- **IDE** — `nexel rebuild-manifest` walks `<skillsDir>/*/SKILL.md`, emits canonical `install.json`. `lint:manifest` becomes regen-and-diff. `ProductConfig.manifestMode: "authored" | "derived"`.
- Removes the double-encoding (frontmatter ↔ manifest) friction.

### Rank 8 — Idea 4b: Multi-tenant install plane + `runTransactional` (Conf 75%, Complexity Med-High)
- **IDE** — `<target>/.nexel/host-registry.json` + new verbs `nexel hosts` / `nexel host gc`. Public `runTransactional({steps, onRollback})` API.
- Distribution-scale enabler. Lower confidence; defer until adoption signal.
