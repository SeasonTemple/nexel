## Changed ‚Äî breaking (safe only pre-adoption)

- **`STATE_DIRNAME` `.skillctl` ‚Üí `.nexel`** (`scripts/installer/core/filesystem.mjs`).
  The on-disk state directory every install / uninstall / update / repair
  / drift operation reads and writes is renamed. **This is NOT a
  zero-behavior change** ‚Äî unlike v0.5.0 (2a) and v0.5.1 (Candidate 1),
  the on-disk contract changes. It is safe **only because the project has
  zero adoption** (unpublished, git-tag/vendor, no known consumers ‚Äî so
  there are no existing `.skillctl/` directories to orphan). **There is
  no migration shim and none is needed, by design.** ADR-0008 supersedes
  ADR-0007 **D4 only** (the freeze rationale and its no-migration-by-
  freeze consequence); ADR-0007's other decisions remain in force.

- For git-tag / vendor consumers: this is a **breaking on-disk change** ‚Äî

  contract clock remains **not started** (ADR-0007 D5); `"private": true`
  retained. No npm publish ‚Äî that is 2b, still deferred.
