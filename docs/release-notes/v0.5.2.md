# v0.5.2

Patch: completes the `skillctl` → `nexel` identity rename. Contains a
**deliberate breaking on-disk-contract change** — read the State
directory note below. Plan
`docs/plans/2026-05-18-006-refactor-residual-skillctl-cleanup-plan.md`;
decision `docs/adr/0008-unfreeze-state-dirname-rename-to-nexel.md`.

## Changed — breaking (safe only pre-adoption)

- **`STATE_DIRNAME` `.skillctl` → `.nexel`** (`scripts/installer/core/filesystem.mjs`).
  The on-disk state directory every install / uninstall / update / repair
  / drift operation reads and writes is renamed. **This is NOT a
  zero-behavior change** — unlike v0.5.0 (2a) and v0.5.1 (Candidate 1),
  the on-disk contract changes. It is safe **only because the project has
  zero adoption** (unpublished, git-tag/vendor, no known consumers — so
  there are no existing `.skillctl/` directories to orphan). **There is
  no migration shim and none is needed, by design.** ADR-0008 supersedes
  ADR-0007 **D4 only** (the freeze rationale and its no-migration-by-
  freeze consequence); ADR-0007's other decisions remain in force.

## Removed

- The misleading npm-version badge from `README.md` / `README.zh-CN.md`.
  It rendered the **unrelated squatted third-party `skillctl` package's**
  version on this project's READMEs — active misinformation. It is **not**
  repointed at `nexel` (unpublished); a correct `nexel` badge is additive
  work deferred to the publish step (2b). `scripts/lint-release-sync.mjs`'s
  comment documenting that badge was updated concomitantly (comment only;
  lint logic unchanged).

## Added

- `docs/adr/0008-unfreeze-state-dirname-rename-to-nexel.md` — records the
  zero-adoption premise that voids ADR-0007 D4's freeze, scoped to
  supersede D4 only.

## Hardening (Tier-2 code-review driven)

- A `STATE_DIRNAME === ".nexel"` guard test (`commands/index.test.mjs`)
  locks the deliberate breaking rename — without it the change was only
  protected by stateDirFor self-consistency, so a typo'd/reverted
  constant would have kept the suite green.
- `.gitignore` now ignores `.nexel/` and `.skillctl/` — guards against a
  maintainer who ran the bin (during or after the skillctl era) from
  accidentally committing a local managed-state dir from this working
  tree (parallels the existing `.netops-agent-skills/` guard).
- ADR-0008's supersession wording corrected: it no longer claims D4 is
  "marked" in ADR-0007 (the repo convention deliberately leaves the
  superseded ADR verbatim; the superseding ADR carries the pointer).

## Explicitly unchanged (by reasoned decision, not omission)

Retention reasons independent of user count: historical records
(`v0.1.0`–`v0.4.0` notes, ADRs `0001`–`0006`, prior plans — rewriting
falsifies history); the product-literal test guards' `skillctl` token
(it is the legacy-leak detector — removing it removes protection);
ADR-0007 + `v0.5.0`/`v0.5.1` notes (they document the rename);
`pre-skillctl` historical comments in `run.mjs`/`dispatch.mjs`;
`lint-release-sync.test.mjs` synthetic fixtures. After this release the
identity rename is functionally complete — remaining `skillctl`
occurrences are 100% by-design.

## Notes

- For git-tag / vendor consumers: this is a **breaking on-disk change** —
  a fresh install writes `.nexel/` instead of `.skillctl/`. Pre-adoption
  by design; no migration is provided or required. The public-API
  contract clock remains **not started** (ADR-0007 D5); `"private": true`
  retained. No npm publish — that is 2b, still deferred.
