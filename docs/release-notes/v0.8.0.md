### Scope decisions recorded at plan time

- **Marketplace JSON sync deferred to follow-up.** 原 brainstorm R12/R13 计划在 `activate` 里同步 `.claude-plugin/plugin.json` / `.codex-plugin/plugin.json` 字段；plan-time 实查现有 sample 文件发现动态内容基本为零（`skills: ["./skills/"]` 是静态目录指针；`version` 同步已由 v0.7 U6 `verifyMarketplaceLockstep` 覆盖）。等真有动态字段浮现时再 re-open。
- **`--target=opencode` 选择 refuse 而非 silent no-op.** OpenCode 的运行时 hook 在 OpenCode 启动时通过 `nexel/adapters/opencode-plugin` 已经 mutate config；`nexel activate --target=opencode` 要么 no-op 要么重复写。Refuse 明确语义比静默更诚实。
- **State-derived scope inheritance deferred to follow-up.** 原 R17 计划从 state.json 推导 scope（user / project）；v0.8 改为 `--scope` 显式（默认 `project`）。每个 adapter 有自己的 state；跨 adapter scope 推导复杂度对 v0.8 不划算。

- **CLAUDE.md / AGENTS.md fence marker contract** — release notes 记一句，避免用户在自家 prose 里手写匹配 marker

### Notes

- `nexel activate` 默认 `--scope=project`（写到 cwd `./CLAUDE.md` / `./AGENTS.md`）；想覆盖 home 加 `--scope=user`
- 已装 OpenCode plugin 的产品无需改任何东西 —— `opencode-instructions:` 别名继续工作；可逐步迁移 skill 到 `host-instructions:`
- 多 product 同 host 共存：fence marker 携带 `productName`，互不覆盖；这是为 Idea 4b 多租户 host-registry 提前对齐的钩子
- Fence marker 约定：`<!-- nexel:<productName>:begin -->` / `<!-- nexel:<productName>:end -->`；用户在自家 `CLAUDE.md` 散文里**不要**手写此模式（不太可能撞，但避免歧义）

### Scope decisions recorded at plan time

- **Marketplace JSON sync deferred to follow-up.** Original brainstorm R12/R13 planned to sync `.claude-plugin/plugin.json` / `.codex-plugin/plugin.json` fields inside `activate`; plan-time inspection of the existing sample files found essentially no dynamic content (`skills: ["./skills/"]` is a static directory pointer; `version` synchronization is already covered by v0.7 U6 `verifyMarketplaceLockstep`). Re-open when real dynamic content emerges.
- **`--target=opencode` is refused rather than silent no-op.** OpenCode's runtime hook already mutates config at OpenCode boot via `nexel/adapters/opencode-plugin`; `nexel activate --target=opencode` would either be a no-op or a double-write. Refusing with a typed error is more honest than silence.
- **State-derived scope inheritance deferred to follow-up.** Original R17 planned to infer scope (user / project) from state.json; v0.8 ships explicit `--scope` (default `project`). Each adapter has its own state; cross-adapter scope inference is too much complexity for v0.8.

### Notes

- `nexel activate` defaults to `--scope=project` (writes to cwd `./CLAUDE.md` / `./AGENTS.md`); pass `--scope=user` for home
- Products already wired with OpenCode plugin runtime need no changes — `opencode-instructions:` alias keeps working; migrate skills to `host-instructions:` at your own pace
- Multi-product coexistence: fence markers carry `productName` so they do not overwrite each other; this is the deliberate hook for Idea 4b's multi-tenant host-registry
- Fence marker convention: `<!-- nexel:<productName>:begin -->` / `<!-- nexel:<productName>:end -->`; do not handwrite this pattern in your own `CLAUDE.md` prose (collision is unlikely but the convention exists for the kernel's own writes)
