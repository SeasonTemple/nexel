# v0.9.0 — backlog drain + 4 ADRs (0016-0019)

## English

### Highlights

v0.9.0 clears the four round-7 backlog items deferred from v0.8.5 and
lands four new ADRs governing each. The release is purely additive to
the public API: no breaking changes, no schema migrations, no
`ProductConfig` identity changes.

### What's new

- **ADR-0016 Pluggable Validator Rule Registry** (PR #10). External
  packages can register custom manifest validation rules via
  `registerValidatorRule(rule)` exported from `installer/index.mjs`.
  Rules emit findings using the existing manifest validator envelope
  (`{severity, section, id, message}`), so they compose with the
  existing `formatFindings` and `exitCodeFor` without a translation
  layer. Rules returning non-array values (Promises, single Finding
  objects, undefined) are quarantined as `parse-error` rather than
  silently dropped — the API does not support async rules in v0.9.0.

- **ADR-0017 deactivate verb** (PR #12, adv-009). `nexel deactivate`
  is the exact-invert counterpart to `nexel activate`. Removes only
  the THIS-product ambient fence from a host file (`CLAUDE.md` /
  `AGENTS.md`); other products' fences are preserved byte-identically.
  Reuses the existing per-product `fenceRegex(productName)`, the
  `<target>.lock` serialization, the adv-004 target-symlink defense,
  and the adv-r5-C lock-symlink defense. Mirrors the activate side's
  unbalanced-marker guard (adv-002) and managed-header sentinel check
  (P1#6) so user prose enclosed in literal nexel-shaped markers cannot
  be silently deleted. Multi-product enumeration (`--all-products`
  flag) is intentionally deferred to a future ADR.

- **ADR-0018 CI bless path for ADR-0013** (PR #8). The mandatory
  pre-tag review gate is now enforceable in CI in addition to the
  maintainer-machine path. `release.yml` gains a `review-gate` job
  that re-runs deterministic checks (`npm test`,
  `release:preflight`, `lint:release-sync`) and a defense-in-depth
  scan for commit messages advertising hook bypasses
  (`--no-verify` / `HUSKY=0` / `core.hooksPath=/dev/null`).
  `verify-release-tag.mjs` accepts either the local `.nexel/`
  marker (with SHA verification) or the CI workflow's
  `NEXEL_CI_BLESSED=true` signal. CI gate is deliberately weaker
  than the local `ce-code-review` mandate — see ADR-0018
  Consequences for the trust framing.

- **ADR-0019 TOCTOU lstat/lockSync cross-platform defense** (PR #11).
  Closes three lstat→mutation TOCTOU windows: ambient-fence write
  target, ambient-fence lock path, and `promoteStagedFiles` staging
  promotion. Uses post-lock re-lstat + inode/dev pinning — uniform
  across Linux, macOS, and Windows with no `process.platform`
  branch in `core/` and no `proper-lockfile` replacement. The new
  helper `core/symlink-safe.mjs` is a kernel internal (not
  exported from `installer/index.mjs`). The lock-path verification
  uses a `lockPathMode` flag that skips ino/dev pinning, because
  proper-lockfile's stale-recovery does `rmdir` + `mkdir` and would
  otherwise produce spurious `ERR_TOCTOU_RACE_DETECTED` for users
  recovering from a crashed prior `nexel activate`. The
  `removeAmbientFence` introduced by PR #12 receives the same
  TOCTOU defense in the same commit (folded in from the originally
  planned Phase 6 cleanup).

### Hardening folded in along the way

- `fix(release): scrub GIT_DIR/GIT_WORK_TREE in husky hooks` —
  husky exports git env vars for hook processes, leaking
  `GIT_DIR` into the spawned `npm test` subprocess and breaking
  `execSync("git rev-parse HEAD", { cwd: <non-git-dir> })`
  expectations downstream. Found during parallel-builder
  dispatch when three of four PR builders hit the same hook
  bug at commit time. `.husky/pre-commit`, `.husky/pre-push`,
  `.husky/pre-push.test.mjs`, and
  `cli/commands/index.mjs::getRepoCommit` now scrub the
  inherited env. No behavior change in the green path.

- All four PRs underwent the mandatory release-time multi-agent
  code review (ADR-0013), and every P0 / P1 finding was fixed in
  Phase 4 before merge. Two P0 findings on the deactivate PR
  (missing unbalanced-marker guard and missing
  `isFenceTrusted` check on `removeAmbientFence`) and one P1
  finding on the TOCTOU PR (stale-lock recovery false positive
  on the lock path) were caught by the review gate and would
  have shipped as live regressions otherwise — concrete
  evidence that the gate continues to earn its keep.

### Surface changes

- New public exports from `scripts/installer/index.mjs`:
  - `registerValidatorRule` (ADR-0016)
  - `listValidatorRules` (ADR-0016)
- New CLI verb: `deactivate` (ADR-0017)
- New error codes:
  - `ERR_TOCTOU_RACE_DETECTED` (ADR-0019)
  - `ERR_DEACTIVATE_OPENCODE_REFUSED` (ADR-0017)
  - `ERR_DEACTIVATE_INVALID_SCOPE` (ADR-0017)
- New env signal honored by `verify-release-tag.mjs`:
  `NEXEL_CI_BLESSED=true` (ADR-0018)
- Removed: nothing. No deprecations.

### Deferred and explicitly out of scope

- `deactivate --all-products` multi-product sweep — needs an on-disk
  product registry; tracked as a future ADR topic.
- Windows CI matrix expansion — ADR-0019 verifies cross-platform
  theoretical correctness via Stats.ino semantics, but no Windows
  CI runs ship in v0.9.0.
- npm publish — ADR-0005 v1.0 clock is independent of this release.

## 中文

### 亮点

v0.9.0 清空 v0.8.5 推迟的四项 round-7 backlog,各自落 ADR。整版纯加性公共
API:无破坏性变更,无 schema 迁移,`ProductConfig` 身份不变。

### 新增

- **ADR-0016 Pluggable Validator Rule Registry** (PR #10):外部包可通过
  `installer/index.mjs` 导出的 `registerValidatorRule(rule)` 注册自定义
  manifest 校验规则。规则使用现有 manifest validator envelope
  (`{severity, section, id, message}`) 发 finding,与现有
  `formatFindings` / `exitCodeFor` 无翻译层组合。返回非数组(Promise / 单
  Finding 对象 / undefined)的规则被隔离为 `parse-error`,而非静默丢弃 —
  v0.9.0 不支持异步规则。

- **ADR-0017 deactivate verb** (PR #12, adv-009):`nexel deactivate` 是
  `nexel activate` 的精确反向。仅移除主机文件中本产品的 ambient fence
  (`CLAUDE.md` / `AGENTS.md`);其它产品的 fence 字节级保留。复用现有
  per-product `fenceRegex(productName)`、`<target>.lock` 序列化、adv-004
  target-symlink 防御、adv-r5-C lock-symlink 防御。同时镜像写路径的
  unbalanced-marker 防护 (adv-002) 和 managed-header sentinel 检查
  (P1#6) — 字面 nexel 标记包裹的用户散文不会被静默删除。多产品枚举
  (`--all-products` flag) 故意推迟到未来 ADR。

- **ADR-0018 CI bless path for ADR-0013** (PR #8):强制 pre-tag review
  gate 现可在 CI 路径而非仅 maintainer 机器上执行。`release.yml` 加
  `review-gate` job,重跑确定性检查(`npm test`、`release:preflight`、
  `lint:release-sync`),并对宣告 hook bypass(`--no-verify` / `HUSKY=0`
  / `core.hooksPath=/dev/null`)的 commit message 做防御性扫描。
  `verify-release-tag.mjs` 接受本地 `.nexel/` marker(SHA 校验)或 CI
  workflow 的 `NEXEL_CI_BLESSED=true` 信号。CI gate 故意比本地
  `ce-code-review` 弱 — trust framing 见 ADR-0018 Consequences。

- **ADR-0019 TOCTOU lstat/lockSync 跨平台防御** (PR #11):关闭三处
  lstat→mutation TOCTOU 窗口:ambient-fence write target、ambient-fence
  lock path、`promoteStagedFiles` staging 推进。采用 post-lock re-lstat +
  inode/dev 锁定 — Linux/macOS/Windows 统一,`core/` 无 `process.platform`
  分支,无 `proper-lockfile` 替换。新辅助模块
  `core/symlink-safe.mjs` 是内核内部(不从 `installer/index.mjs` 导出)。
  lock-path 验证用 `lockPathMode` flag 跳过 ino/dev 锁定,因为
  proper-lockfile 的 stale-recovery 做 `rmdir` + `mkdir` 否则会对从崩溃的
  prior `nexel activate` 恢复的用户产生虚假 `ERR_TOCTOU_RACE_DETECTED`。
  PR #12 引入的 `removeAmbientFence` 在同一 commit 收到同样的 TOCTOU
  防御(从原本计划的 Phase 6 cleanup 合并进来)。

### 顺便强化

- `fix(release): scrub GIT_DIR/GIT_WORK_TREE in husky hooks` — husky
  为 hook 进程导出 git 环境变量,`GIT_DIR` 泄漏进 spawned `npm test`
  子进程,破坏下游对
  `execSync("git rev-parse HEAD", { cwd: <non-git-dir> })` 的预期。
  并行 builder dispatch 时四个 PR builder 中有三个在 commit 时被同一
  hook bug 击中,因此发现。`.husky/pre-commit`、`.husky/pre-push`、
  `.husky/pre-push.test.mjs`、`cli/commands/index.mjs::getRepoCommit`
  现已 scrub 继承的环境。绿色路径无行为变化。

- 四个 PR 全部经过强制 release-time multi-agent code review (ADR-0013),
  Phase 4 在 merge 前修了每个 P0 / P1。deactivate PR 上两个 P0
  (`removeAmbientFence` 缺 unbalanced-marker 防护和
  `isFenceTrusted` 检查) 与 TOCTOU PR 上一个 P1 (lock path 的
  stale-lock recovery 假阳性) 被 review gate 拦截 — 否则会作为线上
  regression 出货。具体证据:gate 仍在挣回自己的位置。

### 公共表面变化

- `scripts/installer/index.mjs` 新增公共导出:
  - `registerValidatorRule` (ADR-0016)
  - `listValidatorRules` (ADR-0016)
- 新 CLI 动词:`deactivate` (ADR-0017)
- 新错误码:
  - `ERR_TOCTOU_RACE_DETECTED` (ADR-0019)

  - `ERR_DEACTIVATE_INVALID_SCOPE` (ADR-0017)
- `verify-release-tag.mjs` 识别的新环境信号:
  `NEXEL_CI_BLESSED=true` (ADR-0018)

### 推迟且明确 out-of-scope

- `deactivate --all-products` 多产品扫除 — 需要 on-disk 产品注册表;作为
  未来 ADR 话题跟踪。
- Windows CI matrix 扩展 — ADR-0019 通过 Stats.ino 语义验证跨平台理论正确
  性,但 v0.9.0 不出 Windows CI 运行。
- npm publish — ADR-0005 v1.0 时钟与本版本独立。
