# v0.6.0 — runtime, locale, baseline, release helpers

Release date: 2026-05-22

## Added

- OpenCode plugin runtime helper exported at `nexel/adapters/opencode-plugin`.
  Products can add installed skill paths to `config.skills.paths` and discover
  skill-declared `opencode-instructions` files for `config.instructions`.
- `opencode-instructions` Skill Metadata validation in skill lint and
  manifest validation. Direct install behavior is unchanged, and the metadata
  is not a Manifest field or new Asset type.
- Sample product OpenCode plugin fixture proving plugin mode without importing
  downstream product content.
- Locale catalogs with explicit `--lang`, `NEXEL_LANG`, and product env
  selection. JSON envelope keys and error codes remain language independent.
- Product-agnostic baseline verification for the sample product, pinned to
  English for deterministic contract-output checks.
- Local release helpers: `npm run release:preflight` and `npm run dist`
  produce preflight checks, tarball, and sha256 checksum without remote
  publication.
- Downstream-scale conformance corpus with generic generated content covering
  large manifests, dependencies, host-specific skills, and OpenCode metadata.
- Safe sample install demo via `node scripts/install-skills.mjs`. The demo
  reuses `examples/sample-product`, previews the install plan, writes to a
  temporary target by default, and includes a non-interactive smoke path:
  `node scripts/install-skills.mjs --demo --yes --json --cleanup`.
- `examples/sample-product/README.md` documenting the product fixture layout,
  visual demo, and direct sample bin usage.
- GitHub Actions CI and tag-driven release workflow. Pushing a `vX.Y.Z` tag now
  validates tag/package/release-note alignment, runs tests and preflight,
  builds `nexel-X.Y.Z.tgz`, and uploads the tarball plus `.sha256` to the
  GitHub Release.
- `docs/release-workflow.md` and `scripts/verify-release-tag.mjs` to make the
  repository release procedure agent-followable and machine-checkable.

## Notes

- `"private": true` remains in place. This release improves local distribution
  artifacts and release discipline, but does not start npm publication.
- Product-private NetOps skills, agents, rules, provider setup, GitLab workflow
  content, and remote publishing identity were not imported.
