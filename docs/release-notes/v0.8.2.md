# v0.8.2 — Round-3 regression fixes + release-gate hardening

> 2026-05-29 · Fix 2 P1s introduced by v0.8.1 + new orphan-marker defense + mandatory pre-tag review gate · 441 / 441 tests passing (+4 vs v0.8.1)

v0.8.1's "round-2 follow-ups" turned out to introduce two new P1s of its own
(sentinel substring bypass + locale-dependent sort breaching the byte-stable
contract). Round-3 closes both, adds a third defense against the orphan-BEGIN
class flagged adversarial-wise during the v0.8.2 pre-commit review, fills the
remaining testing gaps, and hardens the release workflow with a mandatory
multi-agent code-review gate before any future tag is pushed.

## English

### v0.8.1 errata + Round-3 fixes

| Ref | v0.8.1 defect | v0.8.2 fix |
|---|---|---|
| **P1#6-bypass** | `isFenceTrusted` used `.includes(MANAGED_HEADER)` substring match; user prose containing the sentinel literal (v0.8.1 release notes themselves publicized the string) passes the trust check, kernel overwrites user content (live-verified) | `isFenceTrusted` now uses **positional anchor**: sentinel must be the first line of the fence body (immediately after the begin-marker newline). Substring hits no longer count as trust |
| **P2-A locale breach** | `readdir.sort(localeCompare)` without explicit locale follows host `LANG`; `tr_TR` vs `en_US` order `İ` / `ä` differently → fence body diverges byte-wise across CI runners, breaking the byte-stable idempotency contract the release notes themselves claimed (live-verified) | Switched to codepoint comparison `a < b ? -1 : a > b ? 1 : 0`. Fully locale-independent |
| **adv-006 orphan-BEGIN (new P1)** | Append path did not check for unpaired begin/end markers; user prose copying the begin marker from documentation (without a matching end) would let writeAmbientFence append below the orphan, and the next update's lazy regex would span from the orphan to the new end, eating any prose in between (live-verified during the v0.8.2 pre-commit review) | Append path now scans for an unpaired `<!-- nexel:<productName>:begin -->` or end marker. If found, refuses with an actionable error instructing manual cleanup |
| **T-06 false-positive test** | The "host-instructions path-escape" test used `../outside.md`, but `isValidRelativeSkillPath` filters `..` before the path-escape branch — the test passed trivially without exercising the targeted branch | Deleted the misleading test and documented the validator branch as functionally unreachable; v0.8.x cleanup candidate to remove the dead branch entirely |

### Compatibility

| What | Before (v0.8.1) | After (v0.8.2) | Action |
|---|---|---|---|
| `isFenceTrusted` algorithm | substring `.includes(sentinel)` | positional: sentinel must occupy line 1 of fence body | v0.8.1 fences (sentinel actually on line 1) keep passing; user prose with sentinel mid-body now correctly refused |

### Release workflow change (mandatory)

`docs/release-workflow.md` Normal path is now 6 steps instead of 5: step 4 is a mandatory pre-tag `ce-code-review` pass. CLAUDE.md "Mandatory release-time review gate" is the authoritative rule.

Skipping this step in a future release is now a documented process bug. Three consecutive failures proved memory-only enforcement insufficient; the rule now lives in two file-system-level locations the agent reads on every session.

### Deferred (round-4 candidates)

- C3 CRLF inside fence body — editor-touched managed fence may be incorrectly refused (UX edge case, not exploit)
- adv-007 release gate is still exhortative — mechanical enforcement would need a husky pre-push hook gating tag operations; weighed against CI verification already in place
- adv-008 "codepoint" comment technically refers to UTF-16 code-unit — documentation correction
- All other v0.8.0 round-2 deferred items remain on the deferred list (adv-002 fenceRegex orphan-BEGIN amplification, adv-003 concurrent-write race, adv-009 no deactivate verb, AC-03 dual envelope shape, M-04 `--target` collision)

### Notes

- Upgrade v0.8.1 → v0.8.2 keeps existing v0.8.1 fences working (sentinel-on-line-1 satisfies the positional anchor)
- Users who copied the sentinel literal from v0.8.1 release notes into their own prose will get an actionable refusal on next activate — by design; no `--force` bypass
- After 4 ship-without-review fixup cycles (v0.7 / v0.8.0 / v0.8.1 / v0.8.2), this v0.8.2 release is the **first time** ce-code-review ran before commit: pre-commit pass spawned 4 reviewers and surfaced 6 findings (3 P1, 2 P2, 1 P3) that were fixed before this commit was created
