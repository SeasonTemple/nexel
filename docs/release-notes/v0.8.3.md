# v0.8.3 — Mechanical release gate + small fence/validator hardening

> 2026-05-29 · ADR-0013 mandatory pre-tag review gate via husky hook · adv-002 closure on trusted-path · CRLF tolerance + validator dead-branch cleanup · 444 / 444 tests passing

v0.8.3 is the round-4 cleanup pass after a 3-release "ship-without-review →
round-1 fixup" pattern (v0.7.0 / v0.8.0 / v0.8.1) finally broke at v0.8.2's
first successful pre-commit review. The pattern repeated despite three
escalating interventions (agent memory, CLAUDE.md rule, release-workflow
step) — each was declarative. v0.8.3 adds the actual mechanical
enforcement.

### Bless workflow (新工作流)

下次 release 必走：

```sh
# 1-3. (preflight + tests + ce-code-review per CLAUDE.md "Mandatory release-time review gate")

# 4. Commit + tag
git add package.json package-lock.json docs/release-notes/vX.Y.Z.md
git commit -m "release: vX.Y.Z"
git tag -a vX.Y.Z -m "vX.Y.Z"

# 5. Bless (required by husky pre-push hook)
npm run release:bless -- vX.Y.Z

# 6. Push
git push origin main
git push origin vX.Y.Z
```

跳过 step 5 → push 被 hook 拒。**这是设计目标**。

Emergency bypass (legit hotfix when reviewer chain broken)：

```sh
git push --no-verify origin vX.Y.Z
```

Audit 任何 `--no-verify` 使用 in release retrospective。

### Bless workflow (new)

Every future release MUST go through:

```sh
# Steps 1-3: preflight + tests + ce-code-review pass (per CLAUDE.md)

# Step 4: commit + tag
git add package.json package-lock.json docs/release-notes/vX.Y.Z.md
git commit -m "release: vX.Y.Z"
git tag -a vX.Y.Z -m "vX.Y.Z"

# Step 5: bless (REQUIRED — husky pre-push enforces)
npm run release:bless -- vX.Y.Z

# Step 6: push
git push origin main
git push origin vX.Y.Z
```

Skipping step 5 → hook refuses the tag push. This is the design goal.

Emergency bypass for legitimate hotfixes when reviewer chain is broken:

```sh
git push --no-verify origin vX.Y.Z
```

Audit any `--no-verify` use in the release retrospective.
