# v0.3.0

Adds verb-scoped progressive CLI help as a product-agnostic kernel
capability, a kernel-generic agent CLI contract document, and README
accuracy corrections. See `docs/adr/0006` and
`docs/plans/2026-05-18-002-feat-verb-scoped-help-and-agent-cli-contract-plan.md`.

## Added

- **Verb-scoped progressive `--help`.** `<bin> <verb> --help` and
  `<bin> help <verb>` now render a focused, verb-scoped usage block
  instead of the full reference. Bare `help`, `--help` with no verb, and
  unknown verbs still render the unchanged composed full body (the safe
  fallback). `<bin> <verb> help` (verb-first) is intentionally *not* a
  help affordance — `help` is a positional to the verb.
- **`strings.help.verb.*`**: 11 verb-help renderers co-declared inside the
  frozen `strings.help` catalog, parameterized off
  `{ binName, prefix, adapterList }` per the i18n-seam contract — zero
  hardcoded product identity. Downstream locale catalogs can override them
  without forking any kernel module.
- **New public export: `renderHelp`** (`index.mjs`). The shared help-routing
  decision used by `createCli`. Additive; `printHelp` is retained
  unchanged. Pre-publish surface per ADR-0005 — additive for convenience,
  not a frozen-contract commitment (the contract clock has not started).
- **`docs/AGENT-CLI-CONTRACT.md`**: a product-agnostic behavioral contract
  for any `createCli`-derived bin (verbs, exit codes, `--json` envelope,
  non-interactive flags, help-affordance rules). Distinct from and
  complementary to netops's product-private `INSTALL-FOR-AGENTS.md`, which
  ADR-0004 excluded — see `docs/adr/0006`.
- `scripts/installer/cli/help.test.mjs` (`npm run test:help`, also in the
  `test` aggregate): verb-help routing coverage. The
  `examples/sample-product/` E2E suite gains two mandatory witnesses,
  including the `<verb> help` reverse-order assertion.

## Changed

- **README accuracy (both locales).** Install instructions now reflect
  ADR-0005 reality (git tag / git-dependency / vendor; the kernel is not on
  npm — the `skillctl` name is an unrelated third-party package). The
  Public API table lists `renderHelp` and no longer lists the deleted
  `pipeline` (removed in v0.2.0, ADR-0003/0004). The Status line and
  Roadmap no longer contradict the shipped version / ADR-0005. The dynamic
  npm-version badge is left as-is (it cannot drift; ADR-0004 deviation
  note).

## Notes

- For git-tag / vendor consumers: `printHelp`'s behavior and its composed
  full-help body are byte-unchanged. The only public-surface change is the
  additive `renderHelp` export. No migration required.
- `package.json` carries `"private": true`; publication remains deferred
  while the package name is provisional (ADR-0005).
