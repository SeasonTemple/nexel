# v0.7.0 — Netops 经验吸收 + Plugin 安装支持

> 2026-05-25 · 七实施单元 + 两份 ADR + Node engines 提升

## 中文

### Highlights

| 维度 | 改动 | 为什么重要 |
|---|---|---|
| **Kernel 韧性** | State health 双校验、`release-preflight` 加 runtime-deps `engines.node` floor 检查、跨 manifest 版本 `verifyMarketplaceLockstep` helper | 静默腐烂的 state.json、隐藏的 Node 兼容回退、跨平台 plugin manifest 版本漂移 — 三类问题在 v0.7 前要靠运气和 grep 才能抓到 |
| **Plugin 安装支持** | `sample-product/` 长出三平台 plugin layout（Claude / Codex / OpenCode）、`scaffold` opt-in verb 一键生成、SPI v1.2 接 ProductConfig 渲染真实命令 | 下游产品想发 plugin 形态以前要自己摸索 layout；现在 sample-product 是完整可抄范例 + 一行 flag 就能让任意 nexel-derived bin 暴露 scaffolder |
| **契约演化** | ADR-0010（SPI v1.2）+ ADR-0011（sample plugin layout 作 v0.7 reference window） | 把契约决策与窗口期写进 ADR，未来变更点和 deprecation 路径有明文 |
| **Breaking** | `engines.node`：`>=18` → `>=22.22.2` | 修正过度承诺；`write-file-atomic@8` / `@clack/prompts@1.3` 实际下限 Node 22.22.2+ |

| What | Before | After | Action |
|---|---|---|---|

合计：**383 / 383 tests passing**。

### Documentation

- **ADR-0010** — SPI v1.2 widening；记录前向兼容、`SPI_DEFAULTS` 引用与 `Function.length` 契约边界
- **ADR-0011** — sample-product plugin layout 作 v0.7-window downstream reference；记录稳定窗口、layout-stability 义务、与 ADR-0009 OpenCode plugin runtime boundary 关系
- **README** / **README.zh-CN** — Node 徽章与正文同步到 `>=22.22.2`
- **scaffold per-verb help** — `strings.{en,zh}.mjs` 加入 `scaffold` 帮助（仅 `enablePluginScaffolder: true` 暴露）

### Notes

- 装 nexel 被 npm `EBADENGINE` 拒：升 Node 到 22.22.2+（建议 LTS 24.x）
- `verifyMarketplaceLockstep` 触发通常意味着新增 plugin manifest 时漏改 version；改齐即可
- `scaffold` verb 默认未开启；纯 installer 下游 `--help` 看不到该 verb，想发 plugin 时给 `createCli` 加 `enablePluginScaffolder: true`

## English

### Highlights

| Area | Change | Why it matters |
|---|---|---|
| **Kernel resilience** | State-health double-check, `release-preflight` runtime-deps `engines.node` floor check, cross-manifest `verifyMarketplaceLockstep` helper | Silent state.json rot, hidden Node-compat regressions, and cross-platform plugin manifest version drift all needed luck + grep to catch before v0.7 |
| **Plugin install support** | `sample-product/` gains a three-platform plugin layout, opt-in `scaffold` verb generates it from ProductConfig, SPI v1.2 renders real install commands | Downstream products had to figure out plugin layout themselves; now sample-product is a complete reference and any nexel-derived bin can expose the scaffolder with one flag |
| **Contract evolution** | ADR-0010 (SPI v1.2) + ADR-0011 (sample plugin layout as v0.7 reference window) | Contract decisions and stability windows are now written down — future deprecation paths have an anchor |
| **Breaking** | `engines.node`: `>=18` → `>=22.22.2` | Corrects an earlier over-promise; `write-file-atomic@8` / `@clack/prompts@1.3` actually require Node 22.22.2+ |

### Compatibility

| What | Before | After | Action |
|---|---|---|---|
| Node runtime | `>=18` | `>=22.22.2` | Upgrade CI runners / dev boxes to Node 22.22.2 LTS+; npm refuses install on lower |
| ProductConfig | 5 required + 8 optional | 5 required + 13 optional | Existing configs zero-change compatible; add `repositoryUrl` when shipping a plugin |
| SPI version | v1.1 | v1.2 (minor) | Old zero-arg adapter implementations still work (JS ignores extra args); `SPI_DEFAULTS` reference identity and `Function.length` are now explicitly out-of-contract |
| Kernel public API | same as v0.6 | + 3 new exports: `verifyMarketplaceLockstep`, `scaffoldPlugin`, `dispatchScaffold` | Pure additions; existing callers unaffected |

### Added — by Implementation Unit

| Unit | Surface | Tests |
|---|---|---|
| **U1** State health | `assertValidExistingState` / `assertStateSelectionsKnown` helpers + full check on install / export / doctor; structural-only on uninstall / update / repair (preserves recovery semantics) | +11 |
| **U2** Preflight engines | `runtime-deps-engines-compatible` preflight check + exported `rangeMinimum` / `rangeAudit` semver helpers (disjunctive ranges, mixed-open detection) | +7 |
| **U3** ProductConfig metadata | Five optional plugin-metadata fields on ProductConfig with derived defaults; `REQUIRED_FIELDS` unchanged | +15 |
| **U4** SPI v1.2 | `pluginInstallInstructions(productConfig)` accepts ProductConfig; three adapters interpolate fields; lazy-validates missing `repositoryUrl` with a typed error string | +4 |
| **U5** Sample plugin layout | Three-platform plugin manifest layout under `examples/sample-product/`; `package.json.exports` subpath; `main` deliberately stays on the kernel barrel | +8 |
| **U6** Marketplace lockstep | `verifyMarketplaceLockstep(specs)` kernel helper + sample contract test against `package.json` | +12 |
| **U7** Scaffold verb | `<bin> scaffold` opt-in verb (`createCli({enablePluginScaffolder: true})`); pure + argv-adapter exports; `--target` / `--force` / `--no-deps` flags; downstream nexel-dep audit; sandbox guard (refuses non-empty target without `package.json`) | +17 |

Total: **383 / 383 tests passing**.

### Notes

- `EBADENGINE` on install → upgrade Node to 22.22.2+ (Node 24 LTS recommended)
- `verifyMarketplaceLockstep` failures usually mean a plugin manifest version bump was missed — align versions and re-run
- The `scaffold` verb stays hidden from `--help` unless the bin opted in via `enablePluginScaffolder: true`
