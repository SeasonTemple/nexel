# v0.6.1 — automated release workflow

Release date: 2026-05-22

## Added

- GitHub Actions release workflow for `vX.Y.Z` tags. Tagged releases now verify
  tag/package/release-note alignment, run the full test suite, run release
  preflight checks, build the npm tarball, and upload both `.tgz` and `.sha256`
  assets to the GitHub Release.
- Repository release workflow documentation covering normal releases,
  emergency asset repair, and required GitHub Actions permissions.
- Machine-checkable release tag validation via
  `scripts/verify-release-tag.mjs`, included in `npm test`.

## Fixed

- Sample product E2E tests no longer depend on the host machine having real
  agent CLIs installed. The `agents` text-mode test now supplies deterministic
  temporary CLI shims, so local and GitHub Actions runs exercise the same path.

## Notes

- `v0.6.0` remains the initial public release. `v0.6.1` is the first release
  produced through the repository's tag-driven GitHub Actions workflow.
- `"private": true` remains in place; this release does not publish to npm.
