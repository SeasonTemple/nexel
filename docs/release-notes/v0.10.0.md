> 2026-05-30 · SPI v1.3 (`contentPipeline`, ADR-0020) + `doctorProbes` seam · new `rebuild-manifest` dev tool · `@nexel/vendor` MVP scaffold · additive-only, no breaking changes

## 中文

### 亮点

| 维度 | 改动 | 为什么重要 |
|---|---|---|
| **SPI v1.3** | 可选 `contentPipeline?: Stage[]` —— 有序、可叠加的内容变换。单个 `transformAssetContent` 自动晋升为 1 阶段流水线。 | adapter 现在能组合多个内容变换(前缀改写、注解……)而非一个单体 hook。纯加法(ADR-0020)。 |
| **Doctor probes** | adapter 可贡献 `doctorProbes()` 健康检查,fail-soft 折叠进 `doctor`。 | 宿主 CLI 特定的健康项(插件运行时配置等)无需 kernel 硬编码即可出现在 `doctor`。 |

### 新增(按区域)

- **`adapters/spi.mjs`** —— `contentPipeline` 字段、校验(形状、阶段 id 唯一、与 `transformAssetContent` 互斥)、解析(`applyDefaults` 自动晋升)。互斥守卫内置于 `applyDefaults`(任何调用路径都 fail-loud,含公共 `applyAdapterDefaults`)。
- **`core/plan.mjs`、`core/stage-asset.mjs`** —— `applyAdapterTransform` 折叠解析后的流水线,逐阶段 thread Buffer;`ERR_TRANSFORM_FAILED.details.stageId` 标出失败阶段。
- **`cli/commands/index.mjs`** —— `doctor` 折叠 adapter `doctorProbes`,对 throw/非数组/格式错误 fail-soft,并拒绝撞内置 check 名的 probe。
- **`scripts/rebuild-manifest.mjs`** —— 派生/`--check` 模式;`lint-skills.mjs` 改从 `schema.mjs` 取类目枚举(`CATEGORY_SET`)。
- **`examples/vendor-tool/`** —— `@nexel/vendor` MVP(pull/diff/fork、本地路径源、manual-merge 闸、前缀改写、sidecar)。

### 文档

- **ADR-0020** —— contentPipeline SPI v1.3。
- 计划 `2026-05-29-002`(vendor)、`2026-05-29-003`(rebuild-manifest)。

### 已知限制

- **内容流水线阶段必须跨调用纯。** kernel 每个 asset 跑两次流水线(算 dry-run plan hash、再 stage)。返回每次不同字节的阶段(计数器、`Date.now()`、随机)会让 dry-run plan hash 与落盘字节分歧;`state.json` 记录用落盘值,篡改检测仍正确,但 plan 预览会错。这是契约要求,记于 `adapters/README.md`;kernel 运行期不强制纯度。
- **无单阶段输出大小上限。** 跨阶段乘性增长内容的流水线可耗尽内存;保持单阶段输出与输入成比例。
- **`@nexel/vendor` 是脚手架**:仅本地路径源(git/registry 推迟)、2-way diff(无 3-way merge)、仅 `manual-merge` 策略、不做 npm publish(ADR-0005 publish clock)。

## English

### Highlights

| Area | Change | Why it matters |
|---|---|---|
| **SPI v1.3** | Optional `contentPipeline?: Stage[]` — ordered, stackable content transforms. A single `transformAssetContent` auto-promotes to a 1-stage pipeline. | Adapters can compose multiple content transforms (prefix rewrite, annotate, …) instead of one monolithic hook. Strictly additive (ADR-0020). |
| **Doctor probes** | Adapters may contribute `doctorProbes()` health checks, folded into `doctor` fail-soft. | Host-CLI-specific health (plugin-runtime config, etc.) surfaces in `doctor` without the kernel hard-coding it. |
| **rebuild-manifest** | New dev/CI tool (`scripts/rebuild-manifest.mjs`) derives `install.json` skill entries from `SKILL.md` frontmatter; `--check` regen-and-diffs. | Removes the frontmatter↔manifest double-encoding. Adding a skill becomes "create the dir, run rebuild". |
| **@nexel/vendor** | MVP scaffold under `examples/vendor-tool/`: declarative skill vendoring (`pull` / `diff` / `fork`) from a local upstream, with a `.vendored.json` provenance sidecar. | Pre-adoption reference for the sister package. Kernel zero-import (tool-private hash). |

### Compatibility

| Surface | Before | After | Action |
|---|---|---|---|
| Adapter SPI | v1.2 (`transformAssetContent?`) | v1.3 (adds optional `contentPipeline?`) | None — adapters omitting `contentPipeline` behave identically; a single hook auto-promotes. Declaring **both** is a construction-time error. |
| `install.json` formatting | hand-authored | canonical (`rebuild-manifest` form) | None required. The sample manifest was canonicalized (inline→multiline arrays); products opting into the derived workflow run `rebuild-manifest --check` in CI. |
| Public API | — | unchanged | No new permanent `index.mjs` symbol; `rebuild-manifest` is a `scripts/` tool, not a library export. |

### Added — by area

- **`adapters/spi.mjs`** — `contentPipeline` field, validation (shape, unique stage ids, mutual exclusion with `transformAssetContent`), resolution (`applyDefaults` auto-promotion). The mutual-exclusion guard is intrinsic to `applyDefaults` (fails loud on every call path, including the public `applyAdapterDefaults`).
- **`core/plan.mjs`, `core/stage-asset.mjs`** — `applyAdapterTransform` folds the resolved pipeline, threading the Buffer stage→stage; `ERR_TRANSFORM_FAILED.details.stageId` names the failing stage.
- **`cli/commands/index.mjs`** — `doctor` folds adapter `doctorProbes` with fail-soft handling for throwing / non-array / malformed returns, and rejects probe names that collide with reserved built-in check names.
- **`scripts/rebuild-manifest.mjs`** — derive / `--check` modes; `lint-skills.mjs` now sources the category enum from `schema.mjs` (`CATEGORY_SET`).
- **`examples/vendor-tool/`** — `@nexel/vendor` MVP (pull/diff/fork, local-path source, manual-merge gate, prefix rewrite, sidecar).

### Notes

- **Content-pipeline stages must be pure across invocations.** The kernel runs the pipeline twice per asset (hashing the dry-run plan, then staging). A stage that returns different bytes per invocation (counters, `Date.now()`, randomness) makes the dry-run plan hash diverge from the staged bytes; the recorded `state.json` hash uses the staged value, so tamper-detection stays correct, but the plan preview would be wrong. Contract requirement documented in `adapters/README.md`; not enforced at runtime.
- **No per-stage output-size ceiling** — a pipeline that grows content multiplicatively can exhaust memory; keep per-stage output proportional to input.
- **`@nexel/vendor` is a scaffold**: local-path sources only (git/registry deferred), 2-way diff (no 3-way merge), `manual-merge` policy only, no npm publish (ADR-0005 publish clock).
